01 — Scope

Goal & honest scope

The original pitch called for true sodium-specific sweat sensing. Real ion-selective electrode (ISE) sensing needs either a commercial Na⁺-ISE screen-printed electrode (Metrohm DropSens, quote-only pricing) plus an AD5941-class analog front end, or custom membrane fabrication — both too heavy for a student MVP timeline.

The MVP instead delivers a heat-stress risk index, not a literal sodium concentration reading:

  • Skin conductance (GSR) as an electrolyte/sweat-loss proxy — not sodium-selective, confounded by all ions and moisture, a documented limitation throughout
  • Heart rate (PPG) as a cardiovascular strain marker
  • Skin temperature as the direct heat signal
  • Fused into a Physiological-Strain-Index-style risk score (HR + temp, extended here with the GSR/electrolyte channel)
Future work

True Na⁺-ISE sensing is a documented stretch/future-work upgrade path, not built into this MVP.

Hardware architecture

Placement: volar (underside) wrist for all three — more eccrine sweat glands than dorsal wrist, avoids finger placement picking up emotional/stress-sweat instead of thermoregulatory sweat, and sits over the radial artery for PPG.

Electrode selection

GSR modules of this class typically ship with reusable dry metal-plate finger straps, not disposable gel pads. Plan: relocate the stock straps from finger to volar wrist rather than buying new electrodes — zero added cost, already reusable.

Fallback: Mindfield Shop EDA Velcro electrodes (~€17–25), dry and reusable. Do not use disposable adhesive ECG pads — single-use, wrong fit for a repeat-wear wearable. Do not use lab-grade Ag/AgCl electrodes (BIOPAC/NEUROSPEC, $92–140+) — research-instrument pricing, not justified at this scope.

MCU & power

  • Seeed XIAO ESP32-C3 (or S3) — thumb-sized, WiFi+BLE, onboard LiPo charge IC + JST-PH connector, USB-C doubles as charge port and programmer
  • LiPo pouch cell, 3.7V, 150–300mAh, JST-PH 2.0 connector
  • Battery level sensing: 2× 220kΩ voltage divider, LiPo+ to GND, midpoint to A0 — halves 4.2V→2.1V into the 3.3V ADC range, ~9.5µA idle draw
  • Status LED: GPIO D1 → 330Ω → LED → GND — solid while charging, blink below 20%, off otherwise
  • Vibration motor on a GPIO for local real-time alert — works standalone, no phone required

Motion-artifact rejection

MPU6050 (GY-521) — green LED alone improves the PPG signal but does nothing for GSR noise from electrode contact-pressure changes during arm movement. MPU6050 lets firmware flag/discard GSR and PPG readings that coincide with high-acceleration events, and doubles as a free cadence/activity-intensity signal.

Upgrade path

If MPU6050-based motion rejection proves insufficient: swap for a BNO086 (7Semi Mini Breakout), same I2C bus. Onboard sensor fusion + step counter at ~11–14× the cost (₹165 → ₹1,845–2,360), only marginal size difference. Not needed for MVP — reach for this only if real running tests show the gating isn't cutting the noise.

Wearable form factor

Sensors mounted on the inner (volar) face of an elastic/velcro wristband strap, wired to a watch-face electronics module. Module housing: small clear project box — the stacked breakouts + battery + perfboard are bulkier than a custom PCB would be, so a generic box gives fitting headroom. This will read as a maker prototype, not a commercial product — acceptable for MVP.

03 — Firmware

Firmware

Sensor fusion / risk scoring

  • Baseline capture: 30–60s at rest before activity, per sensor
  • During activity: track % rise and slope from baseline for GSR and HR; track absolute skin-temp rise
  • MVP alert logic: fixed threshold on combined rise across the three channels — simple, guaranteed to work, teaches the fundamentals
  • Stretch: small classifier (logistic regression / decision tree) fusing GSR level+slope, HR, skin temp, session duration, trained on the student's own logged sessions

Dual-profile mode — athlete vs. outdoor worker

Same sensor fusion pipeline, different config profile — not a hardware fork.

Calibration

Sweat chloride test strips (athlete/CF market, cheap, off-shelf) used occasionally to anchor "GSR rise of X%" against a real measured value — a defensible, if approximate, risk curve instead of an arbitrary threshold.

Signal accuracy improvements

  1. Reusable electrodes over disposables — see Electrode selection above
  2. Oversample + filter — average multiple ADC reads, moving-average/exponential smoothing to kill motion-spike noise
  3. Temperature-compensate GSR — skin conductance is itself temperature-dependent; use the MAX30205 channel to correct the GSR baseline
  4. Adaptive/rolling baseline — EWMA instead of one fixed rest-reading, catches slow drift over long wear (matters more for the 6–8hr outdoor-worker profile)
  5. Motion-artifact rejection — MPU6050, see above
  6. Clean skin site + consistent placement — alcohol wipe, mark the same volar-wrist spot session to session
  7. Multi-point ground-truth anchoring — calibrate at start, mid, and end of a session, not just once

Stretch goal — chloride-colorimetric checkpoint (v1.5, not core MVP)

Gatorade Gx Sweat Patch contains a silver chloranilate colorimetric zone: sweat chloride reacts with it, producing a purple color whose intensity scales with concentration — chloride correlates tightly with sodium (NaCl), closer to a real sodium proxy than GSR alone. Read electronically with a TCS34725 RGB color sensor (I2C, onboard white LED, shares the existing bus).

Why stretch, not core: not continuous (integrates over the session), recurring consumable cost, no published RGB→concentration formula (needs its own calibration curve), and a mounting problem (needs a housing holding the sensor fixed over the reagent zone all session).

04 — App

App integration — open question

Blocks BLE payload design

What phone does the student test on?

  • Device works standalone without any app — the on-device vibration motor is the MVP's real-time alert path regardless of this decision. App is additive Phase 2, not a blocker.
  • If Android: MIT App Inventor + its BLE extension works, or a PWA using the Web Bluetooth API (Chrome/Edge only) — both low lift.
  • If iPhone: MIT App Inventor's BLE extension is Android-only (unshipped for iOS since ~2020). Web Bluetooth is unsupported in iOS Safari entirely — PWA route is a dead end. Would need Flutter/React Native with a native BLE plugin, or native Swift.
  • Action: confirm the student's phone platform before choosing an app framework. Until then, firmware exposes sensor data over a generic BLE characteristic layout so either path can consume it later without a firmware rewrite.

06 — Validation

Testing / validation plan

  1. Bench test each sensor individually against known references (GSR against a fixed resistor, MAX30101/30105 against a pulse oximeter, MAX30205 against a thermometer)
  2. Baseline capture procedure validation — consistent rest readings across sessions
  3. Live session test: log all three channels + timestamps during actual exercise/heat exposure, compare trend shape against expected physiological pattern
  4. Sweat chloride test-strip calibration anchor points (a few sessions) to sanity-check the GSR-based risk index against a real measurement
  5. Alert threshold tuning based on the above

07 — Honesty

Known limitations

  • GSR is not sodium-selective — a total ionic + moisture conductance proxy. Labeled throughout as "heat-stress risk index," not a sodium concentration reading.
  • Skin temp is not core body temp — accepted tradeoff for a wearable, less accurate than the core-temp input used in clinical PSI.
  • Enclosure will read as a prototype, not a polished product.
  • True Na⁺-ISE sensing path (DropSens electrode + AD5941 AFE) documented as future work, not built in this MVP.
  • Microfluidic sweat-channel (Nix/FLOWBIO-style) explicitly deferred — laser/vinyl cutting, medical adhesive layering, leak/clog debugging is beyond current scope. No off-shelf electronically-interfaceable microfluidic patch exists to buy instead (Macroduct is clinical/bulky, Epicore Discovery Patch is research-quote-only, Gatorade Gx patch is colorimetric-only with no electronic interface).
